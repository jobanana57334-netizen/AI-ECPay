🛠️ 開發流程與技術挑戰 (Development Process & Challenges)
本專案的金流整合經歷了從「AI 輔助開發」到「深度手動除錯」的完整歷程：

1. 金流初步整合 (ECPay Integration)
   起手式：利用 AI 助手（Toggler Chat）將 ECPay 綠界金流模組匯入專案。

串接購物車：成功將金流跳轉邏輯與購物車結帳環節結合，實現使用者點擊「立即付款」後跳轉至綠界測試環境。

2. 突破 Webhook 通訊障礙 (The ngrok Breakthrough)
   面臨困境：發現 ECPay 無法主動將「付款成功」的訊號傳回本地開發環境（localhost），導致資料庫無法自動將訂單狀態從「待付款」變更為「已付款」。

解決方案：導入 ngrok 建立內網穿透隧道，為本機伺服器提供公網 URL。

自動化回傳：透過設定 ReturnURL，成功接收綠界 Webhook 通知並實作自動化寫入付款紀錄，讓使用者在支付後能即時看見成功結果。

3. 跨工具協作與除錯 (Multi-AI Debugging)
   除錯歷程：在 Toggler Chat 執行過程中，出現了連接埠設定錯誤、金流狀態更新邏輯失效等問題。

Gemini 深度輔助：轉向使用 Gemini 進行底層代碼修正，包括：

CORS 保安修復：解決跨網域請求被瀏覽器阻擋的 Header 衝突。

加密演算法優化：精準修正 CheckMacValue 的 SHA256 雜湊邏輯，確保與綠界通訊 100% 正確。

4. UI 優化與訂單管理 (UI & State Management)
   介面增強：在金流穩定後，優化訂單詳情 UI 介面，並加入「返回購物資訊」與「取消訂單」按鈕。

取消訂單策略：

功能限制：考量到電商運營邏輯，系統限制「已付款」訂單無法直接取消。

營運流程：為避免使用者惡意下單後無故退費，取消動作目前設計為引導至「聯繫客服」流程，以人工審核方式處理後續退款作業。
