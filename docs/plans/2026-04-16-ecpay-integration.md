# 計畫：整合綠界科技 (ECPay) 測試金流

## 1. User Story

身為一名消費者，我希望在「花漾生活」選購完鮮花後，能跳轉到綠界支付頁面進行虛擬刷卡測試，確保購物流程完整。

## 2. Spec (規格)

- **進入點**：購物車頁面的「結帳」按鈕。
- **串接技術**：使用 `.agents/skills/ecpay` 中的官方定義。
- **環境變數**：需確認 `.env` 中的 `ECPAY_MERCHANT_ID` 等四項參數已正確讀取。
- **回傳處理**：支付成功後需跳轉回 `BASE_URL` 指定的成功頁面。

## 3. Tasks (任務清單)

- [ ] 檢查 `server.js` 是否已引入 ECPay 相關的 Router。
- [ ] 根據 `.agents/skills/ecpay` 的範例，設定金流封裝參數 (MAC 檢查碼計算)。
- [ ] 在前端 `views` 建立一個簡易的結帳導向頁面。
- [ ] 進行一次完整的虛擬刷卡測試，並確認控制台無噴錯。
